Elevation Data Storage and Manipulation¶

Data types to store elevation data obtained from ARC ASCII files.

See http://resources.esri.com/help/9.3/arcgisengine/com_cpp/GP_ToolRef/Spatial_Analyst_Tools/esri_ascii_raster_format.htm for ARC ASCII format specification. See https://en.wikipedia.org/wiki/Esri_grid for more information.

Plots contours of elevation data (topographic map). The value keyword can be used to plot the elevation gradient or curvature instead.

delaunay()[source]¶

Creates Delaunay triangulation of elevation data. Computed mesh is stored under the name ‘delaunay’.

downsample()[source]¶

Downsample elevation data, masking points that do not contribute to change in curvature.

For example, for a collection of coplanar points (defining a plane), the interior points are masked since they are not necessary to define the plane.

split_cell()[source]¶

Create split-cell triangulation of elevation data.

Split-cell meshes split each square defined by the elevation grid into two right triangles. Computed mesh is stored under the name ‘split-cell’.
